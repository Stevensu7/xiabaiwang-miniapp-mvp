# 《瞎掰王》微信小程序 PRD（MVP）

## 1. 产品目标
打造一个3-8人可快速开局的“瞎掰/唬人”派对桌游小程序：系统出题，玩家提交伪答案并投票找真相；通过“猜中真答案 + 骗到他人”双维度计分，形成高互动和高笑点体验。

## 2. 目标用户
- 18-35岁社交型用户（朋友聚会、宿舍、办公室破冰）
- 喜欢狼人杀/谁是卧底/剧本杀的轻策略玩家
- 对“5分钟上手、15分钟一局”有明确偏好的人群

## 3. 规则基线（参考 Balderdash 类机制）
1) 每轮系统给出题目（冷知识、概念、梗来源）。
2) 所有玩家提交一个“看起来像真的”伪答案。
3) 系统把真答案 + 所有伪答案打乱后展示。
4) 玩家各自投票选择“真答案”。
5) 计分：
   - 投中真答案：+2
   - 你的伪答案骗到别人：每骗1人 +1
   - （MVP不做）自投/重复提交/超时惩罚

## 4. MVP 范围
### 必做
- 创建房间（本地房号）
- 加入玩家（昵称）
- 回合流程：出题 → 伪答案提交 → 混排投票 → 结算
- 实时积分榜
- 题库内置（50-100题）

### 不做（后续）
- 语音房/好友邀请链路
- 用户体系与历史战绩
- 题库UGC审核系统

## 5. 页面与交互
1) 首页：创建/加入房间、规则入口
2) 房间页：玩家列表、房主开局
3) 作答页：题目 + 输入框 + 倒计时
4) 投票页：混排答案卡片、单选提交
5) 结算页：本轮得分、累计榜单、下一轮

## 6. 状态机
`Lobby -> Drafting -> Voting -> Settlement -> NextRound/Finish`
- Drafting 超时：自动提交空答案（后续可优化为系统占位）
- Voting 超时：视为弃票

## 7. 数据结构（MVP）
- Room: roomId, hostId, status, round, players[]
- Player: id, name, score
- Round: questionId, truth, fakeAnswers[{playerId,text}], votes[{voterId,targetIndex}]

## 8. 反作弊与风控
- 提交去重：同轮每人仅1次提交
- 投票匿名：仅在结算时显示被谁骗到
- 文本安全：基础敏感词过滤
- 房主踢人/重开（MVP可先不开放踢人）

## 9. 埋点
- 开局成功率、平均局长、回合完成率
- 人均提交时长、弃票率、复玩率（D1）
- Top题目（互动率/骗分率）

## 10. 验收标准（MVP）
- 3名玩家可完整跑通至少3轮不崩溃
- 计分准确率100%
- 首局上手时间 < 60秒
- H5测试版可在线访问

## 11. 迭代路线
- V1.1：微信登录、好友邀请、战绩页
- V1.2：题库扩展 + 难度分层 + 每日挑战
- V1.3：语音模式、排行榜、赛季机制
